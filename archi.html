    Questions to ask SAAS providers around authentication:

  • How are accounts and passwords created and issued (self registration /manually via an administrative login account)
  • Are administrative accounts forced to use 2FA?
  • How does 2FA work (SMS Text/Google Authenticator)?
  • How many different roles are there?
  • Does the system log off the session after a period of inactivity?
  • Does the system lockout accounts after a number of unsuccessful attempts?
  • What is the self service password reset process in the event the user forgets their password (eg security questions and answers)?
  • If security questions and answers are used, are the questions generated by the user?
  • How many questions are there?
  • Is there a lockout mechanism to prevent brute forcing of security answers(how many guesses before its locked out)?
  • How is the password issued (emailed to the user in plain text, rendered in the application or is a reset link emailed to the user)?
  • What is the password complexity policy? What is the minimum length? What characters are permitted and forbidden? Is the user required to use lowercase, uppercase, digits and special symbols?
  • Does the system provide functionality to prevent the reuse of an old password or password that closely resembles an old password?
  • Where are user login credentials stored? Are passwords at rest salted with a value unique to that specific login and then stored using a cryptographicly strong and slow hash that cannot be reversed (not using deprecated hashing techniques such as MD5 and SHA1) ?
  • If SSO is available is it implemented in a fundamentally secure and robust way using standard Microsoft ADFS or SAML2.0 rather than a proprietary system.
  • Does the system have the ability to capture and stop robotic submissions during access logins?
  • Is there a public API and how does the authentication work?

    • Questions to ask SAAS providers around session management:

  • How does session management work? Are passwords stored in cookies? What is its stored in the cookie? Is the cookie encrypted? What hashing mechanism is used?

    • Questions to ask SAAS providers around the security of the web application:

  • Is HSTS enabled?
  • Is the application protected by a WAF?
  • What version of TLS and ciphers are enabled?

    • Questions to ask SAAS providers around multitenancy:

  • How does the SAAS multi-tenanted application sufficiently isolate customers data from other tenants?
  • How does the SAAS multi-tenanted application sufficiently isolate customer documents/files from other tenants?
  • Does the multi-tenanted SAAS application (code) employ appropriate and robust strategies and controls to enforce isolation of tenants from one another (eg to limit the scope of the applications backend database queries to the tenants own data)
  • How does the SAAS multi-tenanted application restrict any one tenant from monopolizing server servers (which as well as affecting performance for other tenants could be used to exploit vulnerabilties)
  • Is data stored at rest in a multi-tenanted SAAS deployment encrypted using unique encryption keys per tenant?
  • Are database backups in a multi-tenanted SAAS deployment encrypted using unique encryption keys per tenant?
  • What controls are in place to prevent data being restored to the wrong tenant?

    • Questions to ask around ETL process:

  • Is sensitive data transferred over a public (Internet) or private (leased line) link?
  • Is sensitive data stored on an SFTP server?
  • Are data exports encrypted at source before being transferred onto the SFTP server?
  • Are data export files secured with access controls at every stage of the ETL process to prevent viewing or 'siphoning off' of the data?
  • At each stage of the ETL process data should be deleted and the processing time at each stage should be minimised so that data doesn't persist at each stage any longer than necessary. How is this ensured?
  • What strategies and controls are used by the ETL application processing logic to enforce isolation of our data from other customers data?
